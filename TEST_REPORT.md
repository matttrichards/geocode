# geo7 PWA User Story Test Report

Date: 2026-06-21

## Scope

Tested the app against the target behavior:

- User can see their current location and generated geo7 code.
- User can enter a geo7 code manually and jump to it.
- User can click a geo7 code shown in search results.
- User can get directions from current location to a selected geo7 code.
- User can get directions from current location to a manually entered geo7 code.
- User can get directions from current location to a manually entered address.
- User can hand off a calculated route to navigation.

## Errors Found And Fixed

1. Navigate used `window.open`, which can be blocked because it behaves like a popup in PWA/mobile contexts.
   - Fix: changed navigation handoff to same-tab `location.href` Google Maps directions URL.

2. Searching or clicking a new geo7 code while directions were open could leave stale route details visible.
   - Fix: selecting a new place/code now closes the directions panel and clears the previous route result.

3. Long address route summaries were clipped with ellipsis and registered as overflow in mobile layout checks.
   - Fix: route summaries now wrap naturally with `overflow-wrap`.

4. Direction input swap could leave hidden coordinate data as `"undefined"` for manually typed fields.
   - Fix: added dataset copy/delete handling for direction input resolution data.

5. Main search did not directly understand geo7 codes.
   - Fix: main search now resolves `COUNTRY/code` or current-country code input before falling back to address search.
